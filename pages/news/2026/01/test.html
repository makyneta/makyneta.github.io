Dominando o useEffect: Um guia prático para React

Entenda como gerenciar side-effects, limpeza de memória e dependências sem cair em loops infinitos nos seus componentes.

João Silva

12 Outubro, 2023

O hook useEffect é uma das partes mais poderosas do React, mas também uma das mais confusas para quem está a começar. Vamos dissecar como ele funciona na prática.

O que é um Side-Effect?

Em termos simples, um "side-effect" (efeito colateral) é qualquer interação do seu componente com o mundo exterior. Isso inclui:

  • ✦ Fetch de dados (API calls).
  • ✦ Alteração manual do DOM (títulos do documento).
  • ✦ Timers (setTimeout, setInterval).

Sintaxe Básica

O useEffect aceita dois argumentos: uma função de callback e um array de dependências.

JS 
useEffect(() => {
  // Código a ser executado
  document.title = count;

  return () => {
    // Cleanup function (opcional)
  };
}, [count]); // Array de dependências

A Importância do Array de Dependências

Se omitires o array de dependências, o useEffect corre em cada renderização. Isso é raramente o que queres. Se passares um array vazio [], o useEffect corre apenas uma vez, semelhante ao componentDidMount.

"O React garante que o cleanup seja executado antes que o componente seja removido da UI."

Dominar este ciclo de vida é crucial para evitar memory leaks em aplicações complexas, especialmente quando a navegas entre rotas ou componentes desaparecem do ecrã.
